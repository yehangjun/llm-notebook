# LLM Notebook SPEC V2

产品名：Prism
口号：Everything about AI

## 1. 产品目标与边界

### 1.1 目标
- 面向中文用户的 AI 信息聚合与学习笔记。
- 聚合来源是有优质 AI 内容的视频、博客、媒体等网站。
- 用户对 AI 内容做学习笔记，并可以公开分享。
- 本网站聚合信息流未覆盖的内容，用户可以从外部内容网站引入
- 先做 Web 站点，支持账号体系、信息流、收藏、笔记、轻社交。
- 当前重点是可邀请小范围用户试用的最小闭环。
- V1版本已经验证了MVP，这个V2版本是个精简但可正式使用的产品。

### 1.2 非目标（当前阶段）
- 不做原创内容平台。
- 不做复杂推荐算法和复杂运营后台。
- 不做生产级多租户/微服务拆分。

## 2. 产品设计思路

## 2.1 学习笔记
可以看做一个简化版的NotebookLM，特点如下：
- NotebookLM每个笔记对应于一个话题，Prism每一个笔记对应于一个外部内容网站的链接，不支持上传文件等。
- 会用 AI 大模型帮助用户自动分析和总结内容，但不做思维导图、播客等重量操作
- 以用户文字内容的学习笔记记录为主
- 鼓励用户公开分享笔记

## 3. 产品方案设计

## 3.1 认证与账号设计

## 3.1.1 登录入口与页面结构
- 首页右上角有一个账号入口按钮：
  - 未登录：`登录/注册`
  - 已登录：显示`昵称`（若有且不等于ID）否则显示`ID`
- 点击后：
  - 未登录：进入认证页（登录/注册双 Tab）
  - 已登录：进入个人资料页
- 登录/退出登录，导航条中这个账号入口按钮自动发生变化

## 3.1.2 注册与登录功能设计
- 注册（密码制）
  - ID 和 email 均唯一校验，用户输入
  - 密码 与 密码校验
  - 昵称、界面语言（中/英，缺省中）
  - 支持 email 验证码
- 登录（密码制）：支持 ID 或 email 登录
- 个人资料页：ID 和 email 之外都可修改
- 支持退出登录
- 支持登录失败限制（轻量风控）
- 支持忘记密码 / 重置密码
- 后端仍保留可插拔 SSO 路由与 provider（gmail/wechat），但前端不展示入口
- 发信账号为：llm_notebook@163.com

## 3.2 管理系统设计
先实现一个简单的管理系统：
- 系统初始化时候加一个管理员账号，和用户账号系统共享
- 管理员登录之后用户前端界面显示管理入口
- 管理系统目前先实现对用户账号的管理
